# 开发检查清单

> 在开发新功能或修改现有功能时，**必须**逐项检查此清单。

## 开始开发前

- [ ] **查阅 ISSUE_LOG**：阅读所有"已解决"条目，特别关注预防措施
- [ ] **查阅设计文档**：找到相关的 Use Case，理解完整流程
- [ ] **确认测试策略**：
  - 单元测试：可以用 mock 隔离依赖
  - 集成测试：**禁止完全 mock**，必须使用真实对象或部分 mock
  - E2E 测试：必须走完整调用链

## 编写代码时

- [ ] **对照设计文档**：每个 Use Case 步骤是否都有对应实现？
- [ ] **公开方法检查**：设计文档提到的公开方法，实现中是否都存在？
- [ ] **事件监听检查**：UI 组件是否正确订阅了事件？（参考 #005）
- [ ] **数据流向检查**：数据是否通过正确的 API 流动？（参考 #003）

## 编写测试时

- [ ] **Main Scenario**：正常流程是否覆盖？
- [ ] **Alternative Scenarios**：所有异常场景是否覆盖？（参考 #001、#003）
- [ ] **集成测试**：是否使用了真实对象？（参考 #005、#007）
- [ ] **测试隔离**：beforeEach 是否清理了 localStorage？（参考 #003）
- [ ] **边界值**：是否测试了边界情况（如金币恰好等于 cost）？

## 提交代码前

- [ ] **运行完整测试套件**：`npx vitest run`，确认无回归
- [ ] **手动测试**：至少测试一次完整流程
- [ ] **对照 ISSUE_LOG 预防措施**：逐项勾选是否遵守
- [ ] **更新 ISSUE_LOG**：如果有新问题或新经验，立即记录
