## 💰 核心资源

### 金币（Fuel Coins）
- **用途**：比赛入场费、购买氮气
- **获取方式**：答题奖励、每日目标奖励、成就奖励
- **存储**：持久化存储，不清除

### 装备币（Gear Coins）
- **用途**：氮气充能（1装备币 = 1氮气）
- **获取方式**：答对复杂题、连击奖励、每日目标奖励
- **存储**：持久化存储，不清除

### 氮气（Nitro Charges）
- **用途**：比赛中加速
- **获取方式**：用装备币购买（1:1兑换）
- **存储**：持久化存储，不清除

### ~~燃油（Fuel）~~
- **状态**：已删除（v2.0简化）
- **原因**：与金币功能重复，简化游戏机制

## 🎯 答题奖励

### 每套题结构（混合题型）

- **5道简单题**（PIT_BOARD, STRATEGY）：每题答对 **+3 金币**
- **5道复杂题**（RADIO_MSG）：每题答对 **+5 金币**
- **答错**：0奖励

### 套题基础收益

假设一套题全对（10题）：
- 简单题：5 × 3 = **15 金币**
- 复杂题：5 × 5 = **25 金币**
- **合计：40 金币**

### 正确率奖励（装备币）

完成一套题后，根据正确率发放装备币奖励：

| 正确率 | 装备币奖励 | 说明 |
|--------|-----------|------|
| 100%（10/10） | +3 装备币 | 全对奖励 |
| ≥80%（8-9/10） | +2 装备币 | 优秀表现 |
| ≥60%（6-7/10） | +1 装备币 | 及格奖励 |
| <60%（<6/10） | 0 装备币 | 无奖励 |

### 每日目标奖励

| 目标 | 条件 | 奖励 |
|------|------|------|
| 正确率100% | 当日所有套题全对 | +30 金币 |
| 正确率≥80% | 当日整体正确率≥80% | +20 金币 |

**注意**：每日目标奖励每日仅发放一次，取最高奖励。

### 收益示例

#### 场景1：一套题全对
- 基础收益：**40 金币**
- 正确率奖励：**+3 装备币**
- **总收益：40 金币 + 3 装备币**
- ✅ 可跑4圈，可买3个氮气

#### 场景2：一套题答对8题（80%）
- 简单题答对4题：4 × 3 = 12 金币
- 复杂题答对4题：4 × 5 = 20 金币
- 正确率奖励：**+2 装备币**
- **总收益：32 金币 + 2 装备币**
- ✅ 可跑3圈，可买2个氮气

#### 场景3：每天完成3套题，全部答对
- 套题收益：40金币 × 3 = **120 金币**
- 装备币收益：3装备币 × 3 = **9 装备币**
- 每日目标奖励（100%正确率）：**+30 金币**
- **总收益：150 金币 + 9 装备币**
- ✅ 可跑15圈，可买9个氮气

## 🏎️ 比赛消耗

### 入场费（按圈数计算）

- **每圈消耗**：10 金币（所有赛道统一）
- **比赛前选择**：玩家可自主选择跑几圈（1-10圈）
- **总费用**：圈数 × 10 金币

**示例**：
- 上海2D 跑3圈：30 金币
- 蒙特卡洛2D 跑5圈：50 金币
- 银石2D 跑10圈：100 金币

### 氮气使用

- 每次氮气加速：消耗1氮气
- 氮气购买：1装备币 = 1氮气
- 比赛前购买氮气，比赛中使用

### 赛道解锁

- **解锁条件**：根据学习成就解锁（详见[赛道系统](#赛道系统)）
- **无需金币**：赛道解锁不消耗金币
- **自由选择**：玩家可自主选择已解锁的任意赛道

## 🛣️ 赛道系统

### 解锁机制

**原则**：赛道解锁**不消耗金币**，只看学习成就。玩家可自主选择已解锁的赛道。

### 赛道列表

| 赛道ID | 名称 | 类型 | 解锁条件 |
|--------|------|------|----------|
| shanghai-2d | 上海国际赛车场 | 2D | 默认解锁 |
| monaco-2d | 蒙特卡洛街道赛 | 2D | 完成10套题 |
| silverstone-2d | 银石赛道 | 2D | 掌握50个词 |
| shanghai-3d | 上海国际赛车场 3D | 3D | 掌握200个词 |

### 解锁条件说明

- **完成套题**：累计完成的套题总数（`learning.totalQuizzes`）
- **掌握词汇**：累计掌握的词汇数量（`learning.totalWordsMastered`）
- 条件达到后自动解锁，无需手动操作

## 👥 多用户系统

### 用户数据存储

每个用户独立存储，使用用户ID区分：

```javascript
// 旧版（单用户）
wr_game_state
wr_word_progress
wr_daily_stats
wr_quiz_session

// 新版（多用户）
wr_user_list                     // 用户列表：['user_001', 'user_002']
wr_game_state_user_001          // 用户001的游戏状态
wr_word_progress_user_001       // 用户001的单词进度
wr_daily_stats_user_001         // 用户001的每日统计
wr_quiz_session_user_001        // 用户001的答题会话
```

### 用户管理

#### 创建用户
- **入口**：首次打开应用，或用户列表为空时
- **方式**：文本框输入用户名
- **验证**：
  - 用户名长度 2-20 字符
  - 不允许重名
- **初始数据**：
  - 金币：0
  - 装备币：0
  - 氮气：0
  - 已解锁赛道：['shanghai-2d']
  - 已解锁成就：[]

#### 切换用户
- **入口**：首页顶部用户名，点击弹出用户列表
- **操作**：选择其他用户，立即切换
- **数据同步**：切换时自动保存当前用户数据，加载新用户数据

#### 限制
- ❌ 不允许删除用户（防止误删学习记录）
- ❌ 不允许重命名用户（简化逻辑）

### 当前用户标识

```javascript
// 存储当前用户ID
localStorage.setItem('wr_current_user', 'user_001')
```

## 🔄 数据重置

### 重置类型

#### 1. 全部重置（一键清零）

**效果**：
- ✅ 金币 → 0
- ✅ 装备币 → 0
- ✅ 氮气 → 0
- ✅ 学习统计（套数、正确率）→ 0
- ✅ 单词掌握进度 → 清空
- ✅ 成就 → 清空
- ✅ 已解锁赛道 → ['shanghai-2d']
- ✅ 每日进度 → 清空

**入口**：
- 设置页面（需要二次确认）

#### 2. 重置今日学习记录

**效果**：
- ✅ 今日答题统计（套数、正确率）→ 0
- ✅ 今日收益（金币、装备币）→ 0
- ❌ 总金币 → 不变
- ❌ 单词掌握进度 → 不变

**入口**：
- 学习页面（开发调试用）

#### 3. 重置本周学习记录

**效果**：
- ✅ 最近7天答题统计 → 清空
- ❌ 总金币 → 不变
- ❌ 单词掌握进度 → 不变

**入口**：
- 学习页面（开发调试用）

## 🔧 开发者调试

### 隐蔽调试口令

在开发环境（localhost/127.0.0.1）下，浏览器控制台自动加载 `scripts/debug-commands.js`，提供以下命令：

```javascript
// 解锁指定赛道
debugUnlockTrack('monaco-2d');
debugUnlockTrack('silverstone-2d');
debugUnlockTrack('shanghai-3d');
debugUnlockTrack('night-race-3d');

// 添加资源
debugAddFuelCoins(1000);    // 添加金币
debugAddGearCoins(500);     // 添加装备币

// 批量操作
debugUnlockAllTracks();     // 解锁所有赛道
debugResetToDefault();      // 重置为默认状态（保留单词进度）

// 帮助
debugHelp();                // 显示所有可用命令
```

所有命令执行后自动刷新页面。

## 📊 数据结构

### GameState（核心状态）

```javascript
{
  // 资源
  fuelCoins: 0,          // 金币
  gearCoins: 0,          // 装备币
  nitroCharges: 0,       // 氮气

  // 学习统计
  learning: {
    totalWordsSeen: 0,       // 见过的词数
    totalWordsMastered: 0,   // 掌握的词数
    totalQuizzes: 0,         // 完成的套题数
    totalQuestions: 0,       // 回答的总题数
    totalCorrect: 0,         // 正确的题数
    lastPerfectQuiz: false,  // 上次是否全对
  },

  // 每日进度
  daily: {
    lastActiveDate: null,    // 上次活跃日期
    streakDays: 0,           // 连续学习天数
    todayQuizzes: 0,         // 今日套数
    todayFuelCoins: 0,       // 今日金币收益
    todayGearCoins: 0,       // 今日装备币收益
  },

  // 成就系统
  achievements: [],           // 已解锁成就ID列表
  unlockedTracks: ['shanghai-2d'],  // 已解锁赛道ID列表
  selectedTrackId: 'shanghai-2d',   // 当前选择的赛道ID

  // 元数据
  version: 4,                // 数据版本号
}
```

### WordProgress（单词进度）

```javascript
{
  "word": "example",
  "wordId": 123,
  "status": "mastered",      // unlearned | exposed | simple_passed | complex_passed | mastered | forgotten
  "simpleCorrect": true,     // 简单题是否答对
  "complexCorrect": true,    // 复杂题是否答对
  "simpleWrongCount": 0,     // 简单题答错次数
  "complexWrongCount": 0,    // 复杂题答错次数
  "firstSeenDate": "2026-06-14",
  "lastSeenDate": "2026-06-14",
  "masteryDate": "2026-06-14"
}
```

## 🔄 迁移计划

### v1 → v2 迁移步骤

1. **数据备份**：导出旧版数据到JSON文件
2. **数据清理**：清除调试数据（几千金币、已解锁赛道等）
3. **结构调整**：
   - 删除 `fuel` 字段
   - 删除 `upgrades` 字段
   - 添加 `version: 4`
   - 重命名 `fuelCoins` → 保持不变
   - 重命名 `gearCoins` → 保持不变
4. **用户数据分离**：
   - 检测 `wr_user_list` 是否存在
   - 不存在则创建默认用户，迁移旧数据
5. **UI更新**：
   - 添加用户切换组件
   - 移除升级系统UI
   - 移除燃油显示

### 兼容性处理

```javascript
// 迁移脚本示例
function migrateToV4(oldState) {
  return {
    fuelCoins: oldState.fuelCoins || 0,
    gearCoins: oldState.gearCoins || 0,
    nitroCharges: oldState.nitroCharges || 0,
    learning: oldState.learning || {},
    daily: oldState.daily || {},
    achievements: [],
    unlockedTracks: ['shanghai-2d'],  // 重置为默认
    selectedTrackId: 'shanghai-2d',
    version: 4,
  };
}
```

## 🔮 后期功能规划

### 装备币扩展用途（暂不实现）

以下功能预留设计，后期版本可能开放：

#### 1. 引擎升级
- **用途**：提升赛车最高速度
- **消耗**：装备币
- **等级**：1-5级
- **每级提升**：最高速度+5%

#### 2. 车身装饰
- **用途**：个性化赛车外观
- **消耗**：装备币
- **类型**：
  - 涂装（车身颜色/图案）
  - 贴纸（车队标识/赞助商logo）
  - 特效（尾焰/光晕）

#### 3. 轮胎升级
- **用途**：提升过弯速度
- **消耗**：装备币
- **等级**：1-5级
- **每级提升**：过弯速度+3%

#### 4. 氮气系统扩展
- **当前**：1装备币 = 1氮气
- **后期**：
  - 氮气容量升级（可携带更多氮气）
  - 氮气效率升级（每次加速效果更强）

### 实现原则

- 所有升级**仅影响比赛体验**，不影响学习进度
- 升级**不提供竞技优势**（如果是多人竞技模式）
- 装备币获取速度设计需考虑后期升级需求

## 🎯 设计原则

1. **简单优先**：删除不必要的复杂度（燃油、升级）
2. **成就驱动**：赛道解锁看学习成果，不看金币
3. **数据隔离**：多用户数据独立，互不影响
4. **安全重置**：重置操作需二次确认，防止误操作
5. **调试友好**：提供隐蔽调试口令，方便开发测试
6. **后期扩展**：预留装备币用途，为后期功能做准备
