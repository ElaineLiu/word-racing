# 赛道重设计 v2 — 设计文档

> **目的**：让上海与蒙特卡洛两条赛道在玩家体验上形成**截然不同**的驾驶感受。
> **状态**：⏳ 待审核（请打开 `track-v2-prototype.html` 配合查看实际形状）
> **画布扩展**：920×620 → **1400×800**（逻辑坐标，渲染层自适应）

---

## 1. 设计哲学

### 上海 v2 — 高速 F1 GP
**主题词**：**速度、流畅、长直道、大圆弧**
- 模仿真实 F1 上海国际赛车场的核心特征：超长直道 + 高速复合弯 + 一处发卡
- 玩家体验：油门拉满的爽快感，刹车点少但需精确，**最高速度**是核心挑战
- 适合新手熟悉操作、追求圈速

### 蒙特卡洛 v2 — 街道赛
**主题词**：**紧密、技术、零长直、窄到怀疑人生**
- 模仿真实摩纳哥街道赛：**没有真正的直道**，连续弯道，发卡 + S 弯密集
- 玩家体验：必须不停减速、转向、再加速，**容错率极低**
- 适合熟练玩家技术挑战

## 5. 玩家体验差异预期

| 场景 | 上海 v2 | 蒙特卡洛 v2 |
|------|---------|------------|
| 油门按住时间 | 70% | 30% |

| 适合操作风格 | 大胆、追求最高速 | 精细、节奏控制 |

## 6. 实施改造清单

> ⚠️ 这一节是开发要做的事，仅供参考。**等设计审核通过再动手**。

### 代码修改

1. **`config/game-config.js`**
   - `CANVAS_WIDTH: 920` → `1400`
   - `CANVAS_HEIGHT: 620` → `800`

2. **`js/game.js`**
   - `_resizeCanvas()`: `gameAspect = 920/620` → `1400/800`
   - `_resizeCanvas()`: `scale = canvasW / 920` → `canvasW / 1400`

3. **`rendering/render-system.js`**（8 处硬编码 920/620）
   - 全部替换为从 `game-config.js` 引入的 `CANVAS_WIDTH/HEIGHT`

4. **`config/track-registry.js`**
   - `shanghai-2d.waypoints` → 22 点新设计
   - `shanghai-2d.trackWidth: 76` → `90`
   - `monaco-2d.waypoints` → 32 点新设计
   - `monaco-2d.trackWidth: 60` → `50`

5. **`config/game-config.js` 中的 `TRACK.WAYPOINTS` 默认值**
   - 与 shanghai-2d 同步（如果 Track 默认构造仍依赖）

### 影响范围（不动的地方）

- **银石赛道**：waypoints 不变，但视觉上会因画布变大而显得相对较小 — 后续单独优化
- **3D 赛道**：未实现，不受影响
- **测试套件**：可能有几个用了 `TRACK.WIDTH=76` 或 920×620 假设的测试需更新

### 验证步骤

1. 改完后跑 `npx vitest run` 看回归
2. 手动测试：依次选择上海、蒙特卡洛、银石，看赛道形状是否符合预期
3. 检查赛车在新画布尺寸下的可见性、UI 元素布局
