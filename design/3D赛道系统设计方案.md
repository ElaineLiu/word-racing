## 🎯 整体设计意图

### 项目背景

Word Racing是一个"背单词玩赛车"的教育游戏，核心循环是：
```
背单词（答题）→ 获得燃油币/齿轮币 → 购买燃油/升级 → 参加比赛 → 强化学习动机
```

目前已实现：
- 5种答题模式（10题/套，3套/天）
- 3条2D赛道（上海、蒙特卡洛、银石）
- 自适应学习系统（Phase 1-5全部完成）
- 解锁系统（基于学习成就）

### 为什么要做3D赛道

1. **提升游戏吸引力**：3D沉浸体验能显著提升玩家粘性，从而强化学习动机
2. **延长学习周期**：作为高阶解锁内容，给已掌握大量单词的用户提供新目标
3. **拓展学习场景**：通过3D赛道引入"单词强化泡泡"，将学习融入游戏

### 设计目标

| 目标 | 说明 |
|------|------|
| **教育价值优先** | 不破坏"背单词→比赛"核心循环，赛道中的单词复习强化学习效果 |
| **完全隔离2D** | 3D赛道作为独立实现，2D赛道现有效果和机制零影响 |
| **架构一致** | 3D赛道严格实现TrackInterface，复用现有EventBus/GameState |
| **可独立测试** | 提供独立入口，开发期间无需经过答题流程 |
| **渐进交付** | Epic化拆解，每个Epic可独立完成、验证、合并 |

### 核心理念

> **赛道是学习成果的展示场，不是金币的产出地。**

- ✅ 燃油币/齿轮币的**唯一来源**是答题
- ✅ 赛道上的单词泡泡提供**复习强化**，给予速度奖励而非货币
- ✅ 3D赛道是**学习成就的奖励**（掌握200词解锁），不是赚金币的捷径

## 🌟 产品Vision

### 用户体验目标

让玩家在通过答题积累足够学习成就后，进入沉浸式3D赛车体验：

- 🏁 **环形赛道竞速**：1-5圈可选，1玩家 + 3 AI对手，动态排名
- 📚 **单词强化泡泡**：赛道上散布已学未掌握单词，撞击触发复习
- 🎮 **重量感操控**：键盘驱动，非瞬时加速，转弯惯性，高速难控
- 🎥 **多摄像机**：追尾视角 / 驾驶舱视角自由切换
- 🤖 **智能AI**：激进/平衡/保守3种个性，排名动态变化
- 🎨 **低多边形美学**：精致不粗糙的Low-Poly风格，街机氛围

### 关键差异化（vs 2D赛道）

| 维度 | 2D赛道 | 3D赛道 |
|------|--------|--------|
| **视觉** | Canvas 2D俯视图 | Three.js 3D第一/第三人称 |
| **AI对手** | 无 | 3个AI对手 |
| **单词复习** | 无 | 单词泡泡系统 |
| **解锁门槛** | 0-50词 | 200词 |
| **目标受众** | 入门玩家 | 进阶玩家 |
| **教育价值** | 学习动机 | 学习动机 + 复习强化 |

### 技术选型

| 技术 | 选择 | 理由 |
|------|------|------|
| 3D引擎 | Three.js | 社区活跃、轻量、文档完善 |
| 物理 | 自定义（时间基础） | 简化开发、性能可控 |
| 模型 | 程序化生成（低多边形） | 无需外部资源、易调整 |
| 测试 | Vitest（已有） | 与项目一致 |
| AI算法 | Pure Pursuit | 路径跟随经典算法 |

### 学习闭环（3D特有）

```
QUIZ答题学习新词 "accomplish"  →  exposed状态
                ↓
        参加3D比赛
                ↓
     赛道上看到 "accomplish" 泡泡
                ↓
   撞击 → 单词卡片显示 → 记忆强化
                ↓
        exposureCount +1
                ↓
   下次答题时该单词优先出现
                ↓
   答对 → mastery进度+1 → 最终掌握
                ↓
        泡泡不再生成
```

### 关键原则

1. **燃油币/齿轮币**：**仅来自答题**，赛道不产出货币
2. **单词泡泡奖励**：仅给速度提升+曝光记录，不给货币
3. **2D/3D共享**：同一GameState、同一商店、同一升级、同一解锁系统

## 📐 架构原则

### 1. 接口一致性
- `Track3D` 和 `Car3D` 实现 `TrackInterface`
- 共享 `GameState` 单一数据源
- 通过 `EventBus` 发射标准事件

### 2. 2D/3D完全隔离
- 2D相关代码（`js/track.js`, `js/car.js`, `rendering/render-system.js`）**不修改**
- 3D相关代码全部在`3d/`目录下
- 通过`TrackFactory`根据`track.type`分发
- `FeatureFlags.isEnabled('3d-track')`控制总开关

### 3. 渲染架构：Track自渲染模式
- `Track3D.render()` 内部调用 Three.js 渲染器
- `Game`类调用 `track.render()` 时无需感知2D/3D
- Three.js Canvas与2D Canvas通过CSS层叠或切换

### 4. 物理系统：时间基础
- 现有Car物理改为时间基础（`deltaTime`参数）
- 帧率无关，物理更精确
- 向后兼容（默认`deltaTime=1/60`）

### 5. 测试优先（TDD）
- 每个UC配套测试文件
- 单元测试覆盖率 > 80%
- 集成测试验证模块协作

## ⚠️ 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| Three.js学习曲线陡峭 | 高 | 中 | Epic 0先做最小原型，渐进迭代 |
| 性能不达标（<30fps） | 中 | 高 | Epic 8性能优化，提供画质选项 |
| 3D/2D数据状态不一致 | 中 | 高 | 强制共享GameState，禁止3D内部状态 |
| 单词泡泡干扰游戏体验 | 中 | 中 | 卡片显示限1.5秒，不暂停游戏 |
| 单词泡泡破坏经济循环 | 低 | 高 | 严格禁止泡泡产生货币，只给速度奖励 |
| AI行为不自然 | 中 | 低 | 简化AI，使用固定路径+随机扰动 |
| 移动端性能差 | 高 | 中 | 默认低画质，移动端检测自动降级 |
| 物理改为时间基础导致回归 | 中 | 高 | 现有测试必须100%通过，新增对比测试 |

## Epic 6: 单词泡泡系统（核心教育价值）

### 🎬 Vision

**目标**: 实现单词强化泡泡系统，在3D赛道上提供已学未掌握单词的复习机会，记录曝光数据反馈到学习系统，强化记忆但**不破坏核心经济循环**。

### 设计理念

**教育心理学依据**:
1. **间隔重复（Spaced Repetition）**：已学单词在不同场景重复出现，强化长期记忆
2. **情境学习（Contextual Learning）**：在游戏情境中接触单词，降低认知负荷
3. **正反馈循环**：复习行为获得游戏奖励（速度提升），但**不替代答题**

**核心原则**:
- ✅ **保护核心循环**：泡泡**不产生**燃油币/齿轮币
- ✅ **强化学习效果**：增加单词曝光，影响下次答题选词
- ✅ **奖励玩游戏行为**：速度提升让玩家愿意撞击泡泡
- ✅ **数据驱动学习**：曝光数据反馈到 AdaptiveSelector

**学习闭环**:
```
答题学新词 "accomplish" (exposed状态)
    ↓
3D比赛遇到 "accomplish" 泡泡
    ↓
撞击 → 单词卡片 → 速度+10%
    ↓
exposureCount +1
    ↓
下次答题 "accomplish" 优先出现
    ↓
答对 → 推进mastery进度
    ↓
mastered后 → 泡泡不再生成
```

**功能**:
- WordBubbleManager：泡泡生成、检测、记录
- WordBubbleModel：泡泡3D模型
- WordCardDisplay：单词卡片UI
- 与AdaptiveSelector集成

**依赖**: Epic 5

**状态**: ⏸️ 未开始

it('should boost car speed on hit', () => {
  const car = new Car3D(0, 0, 0, scene);
