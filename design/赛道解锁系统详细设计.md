# 赛道系统与解锁机制详细设计

## 1. Use Case 规格说明

### UC-01: 成就解锁检查

**描述**：玩家答题后，系统自动检查是否达成成就条件，解锁相应奖励。

**Actors**：
- 玩家（答题触发）
- 系统（自动检查）

**Preconditions**：
- 玩家已完成至少一套题
- GameState 已初始化
- AchievementManager 已实例化

**Main Scenario**：
1. 玩家在 LearningController 提交最后一题答案
2. LearningController 调用 `sessionManager.submitAnswer()`
3. LearningController 更新 GameState.learning 统计数据
4. LearningController 调用 `achievementManager.checkAll()`
5. AchievementManager 遍历所有成就定义
6. 对于每个未解锁的成就：
   a. 调用成就的 `check(state)` 方法
   b. 如果条件满足，调用 `unlock(achievement)`
7. 解锁过程中：
   a. 将 achievement.id 添加到 `state.achievements`
   b. 如果奖励是赛道，添加到 `state.unlockedTracks`
   c. 如果奖励是金币，增加 `state.fuelCoins`
   d. 发送 `Events.ACHIEVEMENT_UNLOCKED` 事件
8. UI 监听事件，显示成就 Toast 通知

**Alternative Scenarios**：

*A1: 成就已解锁*
- 跳过检查，继续下一个成就

*A2: 条件未满足*
- 不做任何操作，继续下一个成就

*A3: 奖励赛道已解锁*
- 跳过添加，但仍然触发事件通知

**Acceptance Criteria**：
- [ ] AC1: 完成1套题后，自动解锁"first-quiz"成就
- [ ] AC2: "first-quiz"解锁后，`state.unlockedTracks` 包含 'shanghai-2d'
- [ ] AC3: 成就解锁时，UI 显示 Toast 通知
- [ ] AC4: 已解锁的成就不会重复触发
- [ ] AC5: 成就检查不影响答题流程性能（< 50ms）
- [ ] AC6: 测试覆盖：单元测试（check逻辑）、集成测试（LearningController调用）

### UC-02: 赛道选择

**描述**：玩家在商店界面选择要比赛的赛道，系统检查解锁状态和金币余额。

**Actors**：
- 玩家（主动选择）
- Game（状态管理）

**Preconditions**：
- 玩家在 SHOP 状态
- GameState 已加载
- 至少有一条赛道已解锁

**Main Scenario**：
1. 玩家点击商店的"赛道"标签页
2. ShopView 调用 `game.getAvailableTracks()`
3. Game 从 GameState 读取 `unlockedTracks` 和 `fuelCoins`
4. Game 遍历 TRACK_REGISTRY，对每条赛道标记：
   - `unlocked`: 是否在 `unlockedTracks` 中
   - `canAfford`: `fuelCoins >= track.cost`
5. ShopView 渲染赛道列表
6. 玩家点击某条赛道的"选择"按钮
7. ShopView 调用 `game.selectTrack(trackId)`
8. Game 执行：
   a. 检查赛道是否解锁（`unlockedTracks.includes(trackId)`）
   b. 检查金币是否足够（`fuelCoins >= track.cost`）
   c. 保存选择：`gameState.set('selectedTrackId', trackId)`
9. ShopView 显示"已选择"状态

**Alternative Scenarios**：

*A1: 赛道未解锁*
- 步骤7抛出 Error('Track not unlocked')
- ShopView 捕获错误，显示提示"🔒 需要完成成就解锁"

*A2: 金币不足*
- 步骤7抛出 Error('Insufficient fuel coins')
- ShopView 捕获错误，显示提示"🪙 金币不足"

*A3: 赛道不存在*
- 步骤7抛出 Error('Unknown track')
- ShopView 捕获错误，记录日志，回退到默认赛道

**Acceptance Criteria**：
- [ ] AC1: 未解锁的赛道显示🔒图标，点击无反应
- [ ] AC2: 金币不足的赛道按钮 disabled
- [ ] AC3: 选择成功后，GameState.selectedTrackId 更新
- [ ] AC4: 切换赛道后，下次进入商店保持上次选择
- [ ] AC5: 测试覆盖：单元测试（检查逻辑）、集成测试（ShopView调用）

### UC-03: 开始比赛（扣金币）

**描述**：玩家点击"开始比赛"，系统扣除相应金币并进入倒计时。

**Actors**：
- 玩家（触发比赛）
- Game（状态管理）

**Preconditions**：
- 玩家在 SHOP 状态
- 已选择赛道
- 已加满燃油
- 金币足够支付赛道费用

**Main Scenario**：
1. 玩家点击"开始比赛"按钮
2. ShopView 调用 `game.startRace()`
3. Game 执行：
   a. 获取当前选择的赛道 ID
   b. 从 TRACK_REGISTRY 查找赛道配置
   c. 验证金币：`fuelCoins >= track.cost`
   d. 扣除金币：`gameState.modify('fuelCoins', -track.cost)`
   e. 创建赛道实例：
      - 如果 track.type === '2d'：`new Track(waypoints, trackWidth)`
      - 如果 track.type === '3d'：`new Track3D(sceneConfig)`（待实现）
   f. 更新渲染系统：`renderSystem.setTrack(track)`
   g. 设置游戏状态：`this.state = GAME.STATES.COUNTDOWN`
   h. 初始化倒计时：`this.countdownFrames = DISPLAY.COUNTDOWN_FRAMES`
4. 进入 COUNTDOWN 状态
5. 游戏主循环检测状态变更，开始渲染倒计时动画
6. 倒计时结束后进入 RACING 状态

**Alternative Scenarios**：

*A1: 金币不足*
- 步骤3c 验证失败
- 抛出 Error('Insufficient fuel coins')
- ShopView 捕获错误，显示提示"🪙 金币不足"

*A2: 赛道数据错误*
- 步骤3e 创建失败
- 记录错误日志
- 回退到默认赛道 'shanghai-2d'
- 继续执行步骤3f

*A3: 燃油为空*
- 步骤2前，ShopView 检查 `game.fuel === 0`
- 显示提示"⚠️ 请先加满燃油"
- 阻止进入比赛

**Acceptance Criteria**：
- [ ] AC1: 开始比赛后，GameState.fuelCoins 减少对应赛道费用
- [ ] AC2: 金币不足时无法开始比赛
- [ ] AC3: 扣金币后立即触发 Events.COINS_CHANGED 事件
- [ ] AC4: 进入 RACING 状态后，渲染的是选择的赛道
- [ ] AC5: 测试覆盖：单元测试（扣金币逻辑）、集成测试（Game状态流转）

### UC-04: 查看成就列表

**描述**：玩家查看所有成就及其解锁进度。

**Actors**：
- 玩家（主动查看）
- UI（显示）

**Preconditions**：
- 玩家在 MENU 或 QUIZ 状态
- AchievementManager 已初始化

**Main Scenario**：
1. 玩家点击"成就"按钮（在主菜单或答题页）
2. UI 创建成就面板模态框
3. UI 调用 `learningController.getAchievements()`
4. LearningController 调用 `achievementManager.getAllStatus()`
5. AchievementManager 执行：
   a. 读取 `state.achievements` 获取已解锁列表
   b. 遍历 ACHIEVEMENTS，对每个成就：
      - 标记 `unlocked: achievements.includes(id)`
      - 计算 `progress: { current, target }`（如果适用）
   c. 返回完整列表
6. UI 渲染成就列表：
   - 已解锁：显示 🏆 图标，高亮背景
   - 未解锁：显示 🔒 图标，灰色背景

**Alternative Scenarios**：

*A1: 无成就数据*
- 返回空列表
- 显示提示"暂无成就"

*A2: 成就定义更新（新增成就）*
- 自动包含新成就
- 新成就默认未解锁状态

**Acceptance Criteria**：
- [ ] AC1: 已解锁成就显示 🏆 图标和完成时间
- [ ] AC2: 未解锁成就显示进度（如：5/10 套题）
- [ ] AC3: 成就列表按解锁状态和时间排序（已解锁在前）
- [ ] AC4: 测试覆盖：单元测试（getAllStatus逻辑）

## 2. 架构设计

### 2.1 分层架构

```
┌─────────────────────────────────────────────┐
│           UI Layer (Views)                  │
│  ShopView, AchievementToast, MainView       │
└─────────────────┬───────────────────────────┘
                  │ calls
┌─────────────────▼───────────────────────────┐
│        Application Layer (Controllers)      │
│  Game, LearningController                   │
└─────────────────┬───────────────────────────┘
                  │ uses
┌─────────────────▼───────────────────────────┐
│         Domain Layer (Systems)              │
│  AchievementManager, ShopSystem             │
└─────────────────┬───────────────────────────┘
                  │ depends on
┌─────────────────▼───────────────────────────┐
│       Infrastructure Layer (Core)           │
│  GameState, EventBus, Storage               │
└─────────────────────────────────────────────┘
```

**设计原则**：
- 单向依赖：上层依赖下层，下层不依赖上层
- 依赖注入：通过构造函数注入依赖（EventBus, GameState）
- 事件驱动：跨层通信通过 EventBus，避免紧耦合

---

### 2.2 模块职责

#### **config/achievements.js**
- **职责**：定义所有成就配置
- **导出**：`ACHIEVEMENTS` 常量对象
- **类型**：纯数据配置，无逻辑

#### **config/track-registry.js**
- **职责**：定义所有赛道配置
- **导出**：`TRACK_REGISTRY` 常量对象
- **类型**：纯数据配置，无逻辑

#### **systems/achievement-manager.js**
- **职责**：
  - 检查成就条件
  - 解锁成就并发放奖励
  - 提供成就状态查询
- **依赖**：EventBus, GameState
- **公开方法**：
  - `checkAll()` - 检查所有成就
  - `getAllStatus()` - 获取所有成就状态
- **私有方法**：
  - `#unlock(achievement)` - 解锁单个成就
  - `#unlockTrack(trackId)` - 解锁赛道
  - `#getProgress(achievement, state)` - 计算进度

#### **core/game-state.js**（扩展）
- **新增字段**：
  - `achievements: string[]` - 已解锁成就ID列表
  - `unlockedTracks: string[]` - 已解锁赛道ID列表
  - `selectedTrackId: string` - 当前选择的赛道ID
  - `learning.lastPerfectQuiz: boolean` - 上次是否全对
- **新增方法**：
  - 无（使用现有的 `get/set/modify`）

#### **js/game.js**（改动）
- **新增方法**：
  - `selectTrack(trackId)` - 选择赛道（带验证）
  - `startRace()` - 开始比赛（扣金币+创建赛道）

- **改动方法**：
  - `constructor()` - 初始化 selectedTrackId

#### **learning/learning-controller.js**（改动）
- **新增依赖**：AchievementManager
- **改动方法**：
  - `constructor()` - 实例化 AchievementManager
  - `submitAnswer()` - 答题后调用 `achievementManager.checkAll()`
  - `completeQuiz()` - 标记 `lastPerfectQuiz`

#### **views/shop-view.js**（改动）

### 2.3 游戏状态机联动

**现有状态机**：
```
MENU → QUIZ ↔ SHOP → (COUNTDOWN → RACING → RESULTS) → QUIZ
```

**新增强制检查点**：

```
SHOP 状态进入时：
  → 检查燃油是否足够（fuel > 0）
  → 检查金币是否足够（fuelCoins >= selectedTrack.cost）

COUNTDOWN 状态进入前：
  → 扣除赛道金币
  → 创建赛道实例
  → 验证数据完整性

RACING 状态结束后：
  → 回到 QUIZ（不自动检查成就，等下次答题）
```

**状态转换表**：

| 当前状态 | 触发事件 | 前置条件 | 后置状态 | 副作用 |
|---------|---------|---------|---------|--------|
| MENU | 点击"开始" | - | QUIZ | - |
| QUIZ | 点击"商店" | - | SHOP | - |
| SHOP | 点击"选择赛道" | track unlocked & canAfford | SHOP | 更新 selectedTrackId |
| SHOP | 点击"开始比赛" | fuel > 0 & fuelCoins >= cost | COUNTDOWN | 扣金币，创建赛道 |
| COUNTDOWN | 倒计时结束 | - | RACING | 开始游戏循环 |
| RACING | 完成比赛 | - | RESULTS | 计算分数 |
| RESULTS | 点击"继续" | - | QUIZ | - |

## 3. 接口与类设计

### 3.1 AchievementManager 类

```javascript
/**
 * AchievementManager - 成就管理器
 *
 * 职责：检查成就条件、解锁成就、发放奖励
 */
export class AchievementManager {
  #eventBus;      // EventBus 实例
  #gameState;     // GameState 实例

  /**
   * @param {EventBus} eventBus
   * @param {GameState} gameState
   */
  constructor(eventBus, gameState) {
    if (!eventBus) throw new Error('EventBus is required');
    if (!gameState) throw new Error('GameState is required');

    this.#eventBus = eventBus;
    this.#gameState = gameState;
  }

  /**
   * 检查所有成就（答题后调用）
   * @public
   */
  checkAll() {
    const state = this.#gameState.getAll();
    const unlocked = state.achievements || [];

    for (const [id, achievement] of Object.entries(ACHIEVEMENTS)) {
      if (unlocked.includes(id)) continue;

      try {
        if (achievement.check(state)) {
          this.#unlock(achievement);
        }
      } catch (error) {
        console.error(`Failed to check achievement ${id}:`, error);
      }
    }
  }

  /**
   * 获取所有成就状态（供 UI 显示）
   * @public
   * @returns {Array<Object>} 成就列表，包含解锁状态和进度
   */
  getAllStatus() {
    const state = this.#gameState.getAll();
    const unlocked = state.achievements || [];

    return Object.values(ACHIEVEMENTS)
      .map(ach => ({
        ...ach,
        unlocked: unlocked.includes(ach.id),
        progress: this.#getProgress(ach, state)
      }))
      .sort((a, b) => {
        // 已解锁的排前面
        if (a.unlocked !== b.unlocked) return b.unlocked ? 1 : -1;
        return 0;
      });
  }

  /**
   * 解锁成就
   * @private
   * @param {Object} achievement
   */
  #unlock(achievement) {
    const state = this.#gameState.getAll();

    // 避免重复解锁
    if (state.achievements.includes(achievement.id)) {
      return;
    }

    // 记录成就
    state.achievements.push(achievement.id);
    this.#gameState.set('achievements', state.achievements);

    // 发放奖励
    if (achievement.reward.track) {
      this.#unlockTrack(achievement.reward.track);
    }
    if (achievement.reward.fuelCoins) {
      this.#gameState.modify('fuelCoins', achievement.reward.fuelCoins);
    }

    // 发送事件
    this.#eventBus.emit(Events.ACHIEVEMENT_UNLOCKED, { achievement });

    console.log(`🏆 Achievement unlocked: ${achievement.name}`);
  }

  /**
   * 解锁赛道
   * @private
   * @param {string} trackId
   */
  #unlockTrack(trackId) {
    const state = this.#gameState.getAll();

    if (!state.unlockedTracks.includes(trackId)) {
      state.unlockedTracks.push(trackId);
      this.#gameState.set('unlockedTracks', state.unlockedTracks);
      console.log(`🏁 Track unlocked: ${trackId}`);
    }
  }

  /**
   * 获取成就进度
   * @private
   * @param {Object} achievement
   * @param {Object} state
   * @returns {Object|null} { current, target }
   */
  #getProgress(achievement, state) {
    // 提取数字目标（如："完成 10 套题" → 10）
    const match = achievement.description.match(/(\d+)/);
    if (!match) return null;

    const target = parseInt(match[1]);

    // 根据成就类型确定当前值
    if (achievement.id.includes('quiz')) {
      return { current: state.learning.totalQuizzes, target };
    }
    if (achievement.id.includes('word')) {
      return { current: state.learning.totalWordsMastered, target };
    }

    return null;
  }
}
```

### 3.2 Game 类改动

```javascript
// js/game.js

import { TRACK_REGISTRY } from '../config/track-registry.js';

export class Game {
  constructor(canvas) {
    // ... 现有代码 ...

    // 新增：加载上次选择的赛道
    this.selectedTrackId = this.#gameState.get('selectedTrackId') || 'shanghai-2d';
  }

  /**
   * 选择赛道（带验证）
   * @param {string} trackId
   * @throws {Error} 如果赛道未解锁或金币不足
   */
  selectTrack(trackId) {
    const track = TRACK_REGISTRY[trackId];
    if (!track) {
      throw new Error('Unknown track');
    }

    // 检查是否解锁
    const unlocked = this.#gameState.get('unlockedTracks') || [];
    if (!unlocked.includes(trackId)) {
      throw new Error('Track not unlocked');
    }

    // 检查金币
    const fuelCoins = this.#gameState.get('fuelCoins');
    if (fuelCoins < track.cost) {
      throw new Error('Insufficient fuel coins');
    }

    // 保存选择
    this.selectedTrackId = trackId;
    this.#gameState.set('selectedTrackId', trackId);

    console.log(`Track selected: ${track.name}`);
  }

  /**
   * 开始比赛（扣金币+创建赛道）
   * @throws {Error} 如果金币不足
   */
  startRace() {
    const trackId = this.selectedTrackId;
    const track = TRACK_REGISTRY[trackId];

    if (!track) {
      console.error(`Track not found: ${trackId}, fallback to default`);
      this.selectedTrackId = 'shanghai-2d';
      return this.startRace();
    }

    // 验证金币（双重检查）
    const fuelCoins = this.#gameState.get('fuelCoins');
    if (fuelCoins < track.cost) {
      throw new Error('Insufficient fuel coins');
    }

    // 扣除金币
    this.#gameState.modify('fuelCoins', -track.cost);

    // 创建赛道实例
    try {
      if (track.type === '2d') {
        this.track = new Track(track.waypoints, track.trackWidth);
      } else if (track.type === '3d') {
        // TODO: Phase 5 实现
        throw new Error('3D track not implemented yet');
      }

      // 更新渲染系统
      this._renderSystem.setTrack(this.track);

      // 重置赛车位置
      this.car.reset(this.track.startPos.x, this.track.startPos.y, this.track.startPos.angle);

      // 开始倒计时
      this.state = GAME.STATES.COUNTDOWN;
      this.countdownFrames = DISPLAY.COUNTDOWN_FRAMES;

      console.log(`Race started on: ${track.name}, cost: ${track.cost} fuel coins`);
    } catch (error) {
      console.error('Failed to create track:', error);
      // 回退到默认赛道
      this.selectedTrackId = 'shanghai-2d';
      return this.startRace();
    }
  }

  /**
   * 获取可用赛道列表
   * @returns {Array<Object>}
   */
  getAvailableTracks() {
    const unlocked = this.#gameState.get('unlockedTracks') || [];
    const fuelCoins = this.#gameState.get('fuelCoins');

    return Object.values(TRACK_REGISTRY).map(track => ({
      ...track,
      unlocked: unlocked.includes(track.id),
      canAfford: fuelCoins >= track.cost
    }));
  }
}
```

## 8. 风险与对策

### 风险1: 存档迁移失败
**概率**: 低
**影响**: 中
**对策**:
- 使用默认值填充缺失字段
- 添加版本检查和迁移日志
- 提供"重置游戏"选项

### 风险2: 成就检查性能问题
**概率**: 低
**影响**: 低
**对策**:
- 使用早期返回优化
- 避免在 check 函数中进行复杂计算
- 监控 checkAll() 执行时间

### 风险3: 赛道数据错误导致崩溃
**概率**: 中
**影响**: 高
**对策**:
- 添加数据验证
- 使用 try-catch 包裹创建逻辑
- 提供回退机制（fallback to default track）

### 风险4: 金币经济失衡
**概率**: 中
**影响**: 中
**对策**:
- 先用保守数值（高消耗，低奖励）
- 添加数据分析日志
- 准备调整方案
