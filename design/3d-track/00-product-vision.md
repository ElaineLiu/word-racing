## 🎯 整体设计意图

### 项目背景

Word Racing是一个"背单词玩赛车"的教育游戏，核心循环是：
```
背单词（答题）→ 获得燃油币/齿轮币 → 购买燃油/升级 → 参加比赛 → 强化学习动机
```

目前已实现：
- 5种答题模式（10题/套，3套/天）
- 3条2D赛道（上海、蒙特卡洛、银石）
- 自适应学习系统（Phase 1-5全部完成）
- 解锁系统（基于学习成就）

### 为什么要做3D赛道

1. **提升游戏吸引力**：3D沉浸体验能显著提升玩家粘性，从而强化学习动机
2. **延长学习周期**：作为高阶解锁内容，给已掌握大量单词的用户提供新目标
3. **拓展学习场景**：通过3D赛道引入"单词强化泡泡"，将学习融入游戏

### 设计目标

| 目标 | 说明 |
|------|------|
| **教育价值优先** | 不破坏"背单词→比赛"核心循环，赛道中的单词复习强化学习效果 |
| **完全隔离2D** | 3D赛道作为独立实现，2D赛道现有效果和机制零影响 |
| **架构一致** | 3D赛道严格实现TrackInterface，复用现有EventBus/GameState |
| **可独立测试** | 提供独立入口，开发期间无需经过答题流程 |
| **渐进交付** | Epic化拆解，每个Epic可独立完成、验证、合并 |

### 核心理念

> **赛道是学习成果的展示场，不是金币的产出地。**

- ✅ 燃油币/齿轮币的**唯一来源**是答题
- ✅ 赛道上的单词泡泡提供**复习强化**，给予速度奖励而非货币
- ✅ 3D赛道是**学习成就的奖励**（掌握200词解锁），不是赚金币的捷径

## 🌟 产品Vision

### 用户体验目标

让玩家在通过答题积累足够学习成就后，进入沉浸式3D赛车体验：

- 🏁 **环形赛道竞速**：1-5圈可选，1玩家 + 3 AI对手，动态排名
- 📚 **单词强化泡泡**：赛道上散布已学未掌握单词，撞击触发复习
- 🎮 **重量感操控**：键盘驱动，非瞬时加速，转弯惯性，高速难控
- 🎥 **多摄像机**：追尾视角 / 驾驶舱视角自由切换
- 🤖 **智能AI**：激进/平衡/保守3种个性，排名动态变化
- 🎨 **低多边形美学**：精致不粗糙的Low-Poly风格，街机氛围

### 关键差异化（vs 2D赛道）

| 维度 | 2D赛道 | 3D赛道 |
|------|--------|--------|
| **视觉** | Canvas 2D俯视图 | Three.js 3D第一/第三人称 |
| **AI对手** | 无 | 3个AI对手 |
| **单词复习** | 无 | 单词泡泡系统 |
| **解锁门槛** | 0-50词 | 200词 |
| **目标受众** | 入门玩家 | 进阶玩家 |
| **教育价值** | 学习动机 | 学习动机 + 复习强化 |

### 技术选型

| 技术 | 选择 | 理由 |
|------|------|------|
| 3D引擎 | Three.js | 社区活跃、轻量、文档完善 |
| 物理 | 自定义（时间基础） | 简化开发、性能可控 |
| 模型 | 程序化生成（低多边形） | 无需外部资源、易调整 |
| 测试 | Vitest（已有） | 与项目一致 |
| AI算法 | Pure Pursuit | 路径跟随经典算法 |
