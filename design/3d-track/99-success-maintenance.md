## 🔄 文档维护规范

### 更新时机
- ✅ **开始Epic时**：将Epic状态改为 🚧
- ✅ **开始Use Case时**：将UC状态改为 🚧
- ✅ **完成Use Case时**：勾选所有AC，状态改为 ✅
- ✅ **完成Epic时**：勾选所有DoD，状态改为 ✅
- ✅ **设计变更时**：**先改文档，再改代码**

### 命名约定
- Epic/UC状态：⏸️ 未开始 / 🚧 进行中 / ✅ 已完成
- AC格式：`- [ ]` Markdown任务列表

### Git提交规范
```
docs(epic-N): update Epic N status to in-progress
feat(epic-N): complete UC N.X - <feature>
test(epic-N): add unit tests for <feature>
docs(epic-N): mark Epic N as completed
```

### 防止文档过时
- 每次PR必须更新文档相应状态
- 代码review时检查文档同步
- 每个Epic完成后必须有"文档完成"checkbox勾选
