## 一、技术选型确认

### 1.1 核心技术栈
- **语言**: 纯 JavaScript（无框架）
- **渲染**: Canvas 2D API
- **数据持久化**: localStorage
- **构建工具**: 无（纯静态文件，直接浏览器运行）

### 1.2 技术选型理由
- ✅ **纯 JS 优势**: 无需编译、部署简单、适合教育场景
- ✅ **Canvas 渲染**: 高性能 2D 渲染，适合赛车游戏
- ✅ **localStorage**: 轻量级持久化，满足排行榜和进度保存需求
- ✅ **无框架**: 减少依赖，降低维护成本

## 四、关键流程设计

### 4.1 双货币奖励流程
```
答对题目
  ├─ 简单题（选择题）
  │   ├─ 奖励：燃油币（+10 × 难度倍率）
  │   └─ 连击奖励：3题 +5，5题 +10
  │
  └─ 复杂题（阅读理解）
      ├─ 奖励：装备币（+30 × 难度倍率）
      └─ 连击奖励：3题 +20，5题 +50
```

### 4.2 商店购买流程
```
点击商品
  ├─ 检查货币类型（燃油币 or 装备币）
  ├─ 检查货币是否足够
  ├─ 检查购买限制（燃油上限、前置要求）
  ├─ 扣除对应货币
  └─ 添加商品到库存 / 升级改装
```

### 4.3 改装性能计算流程
```
获取改装等级
  ├─ 引擎等级 → 燃油消耗倍率、加速度倍率
  ├─ 轮胎等级 → 转弯速度倍率、off-track 摩擦倍率
  └─ 车身等级 → 最高速度倍率、Nitro 持续时间倍率
  
计算最终性能
  ├─ fuelPerLap = baseFuelPerLap × engine.fuelConsumption
  ├─ maxSpeed = baseMaxSpeed × body.maxSpeed
  ├─ acceleration = baseAcceleration × engine.acceleration
  └─ turnSpeed = baseTurnSpeed × tire.turnSpeed
```

### 4.4 核心状态机（更新后）
```
MENU → QUIZ（选择题目类型、套数）→ SHOP（购买燃料/装备/改装）↔ RACE
  ↑        ↓                        ↓                        ↓
  └────────────────────────────────┴────────────────────────┘
           （比赛结束 / 中途退出，返回 QUIZ）
```

### 6.2 新增文件：upgrade.js
```javascript
/**
 * UpgradeSystem - 改装系统模块
 * 管理改装等级、计算性能参数
 */
class UpgradeSystem {
    constructor(game) {
        this.game = game;
        this.config = { /* 见 3.3 节 */ };
    }
    
    /**
     * 获取当前性能参数
     */
    getPerformance() {
        const e = this.game.upgrades.engine;
        const t = this.game.upgrades.tire;
        const b = this.game.upgrades.body;
        
        return {
            fuelConsumption: this.config.engine[e].fuelConsumption,
            acceleration: this.config.engine[e].acceleration,
            maxSpeed: this.config.body[b].maxSpeed,
            turnSpeed: this.config.tire[t].turnSpeed,
            nitroDuration: this.config.body[b].nitroDuration,
            nitroBoost: this.config.body[b].nitroBoost
        };
    }
    
    /**
     * 升级改装（消耗装备币）
     */
    upgrade(part, level) {
        const config = this.config[part][level - 1];
        
        // 检查装备币是否足够
        if (this.game.gearCoins < config.cost) {
            return { success: false, message: '装备币不足！' };
        }
        
        // 检查前置要求
        if (config.prerequisite && this.game.upgrades[part] < config.prerequisite) {
            return { success: false, message: '需要先升级到 T' + config.prerequisite };
        }
        
        // 扣除装备币
        this.game.gearCoins -= config.cost;
        
        // 提升等级
        this.game.upgrades[part] = level;
        
        return { success: true, message: '升级成功！' };
    }
}
```

### 6.4 风险点与规避措施

| 风险点 | 影响 | 规避措施 |
|--------|------|----------|
| 双货币系统复杂度高 | 可能导致代码混乱 | 严格区分货币类型，新增明确的方法（buyFuel / buyGear） |
| 改装系统性能计算复杂 | 可能导致赛车性能异常 | 单元测试性能计算函数，确保数值正确 |
| 经济平衡难以把握 | 可能导致游戏太难或太简单 | 多次测试，记录货币产出/消耗，动态调整数值 |
| 词汇库不足 | 复杂题无法生成 | 优先扩展 words.json，增加复杂题数据 |
| 代码耦合度高 | 难以维护和扩展 | 新增 upgrade.js 独立模块，降低耦合 |
